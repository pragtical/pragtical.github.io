## Bug Fix

Immediately following v3.8.6 we make another release to fix an issue that
was introducing while trying to save some cpu cycles on the View update routine.

Previously we added a change to prevent unnecessary scroll logic on a View
update but didn't noticed this affected the StatusView show message functionality.
This update change should fix the scenario where a view is marked as non-scrollable
but still hacks around the scroll mechanism as it is done with the StatusView
show_message mechanism which uses the scroll system to display temporary messages.

Downloads on [GitHub](https://github.com/pragtical/pragtical/releases/tag/v3.8.7).
